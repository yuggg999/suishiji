## 更新内容

### 修复：后台通知监听功能失效

**问题描述：**
v1.3.1 版本中，后台通知监听功能失效，无法自动识别支付宝/微信支付通知。

**根本原因：**
- release 构建开启了 ProGuard/R8 混淆（`minifyEnabled true` + `shrinkResources true`）
- `proguard-rules.pro` 只保留了数据库类，未保护 service 包
- R8 将 `PaymentNotificationService` 重命名/混淆后，AndroidManifest 中的引用失效
- 系统无法绑定通知监听服务，导致静默失败

**解决方案：**
1. 在 `proguard-rules.pro` 中添加保护规则：
```proguard
-keep class com.suishiji.service.** { *; }
```

2. 在 `build.gradle` 中添加 release 签名配置：
```groovy
signingConfigs {
    release {
        storeFile file('C:\\Users\\yu\'vv\'y\\.android\\debug.keystore')
        storePassword 'android'
        keyAlias 'AndroidDebugKey'
        keyPassword 'android'
    }
}
```

### 新增：后台定位功能（高德SDK）

**功能说明：**
- 在后台记录交易时，自动获取当前位置信息
- 使用高德SDK进行定位，支持中国地区高精度定位
- 前台服务通知（Android 8.0+ 要求）

**技术实现：**
1. **高德SDK集成**：使用 `AMapLocationClient` 进行单次定位
2. **前台服务**：添加 `NotificationChannel` 和前台服务通知
3. **权限请求**：在 MainActivity 中请求后台位置权限
4. **超时机制**：10秒超时，避免无限等待

**权限要求：**
- `ACCESS_FINE_LOCATION`：精确定位
- `ACCESS_COARSE_LOCATION`：粗略定位
- `ACCESS_BACKGROUND_LOCATION`：后台定位（Android 10+）
- `FOREGROUND_SERVICE`：前台服务
- `POST_NOTIFICATIONS`：通知权限（Android 13+）

**耗电影响：**
- 每次定位约 0.1-0.2% 电量
- 每天 10-20 次交易：总计约 2-5% 电量
- 远低于社交/支付类应用
